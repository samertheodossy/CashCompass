Estimate multipliers and open decisions

These are the main reasons calendar time can exceed focused engineering time.

Apps Script runtime and permissionsAuthorization, quotas, deployment identities, locks, eventual consistency, and account-specific Drive visibility can add investigation cycles. The intentional drive.file scope means populated test fixtures must be Central-created rather than merely shared to the test account.
Disposable fixture privacyHarness fixtures must default to Restricted sharing, fail closed if link-accessible, contain only synthetic data, and return to Trash without affecting owner or bounded workbooks.
Real workbook safetyDisposable tests are fast; proving mature bounded workbooks remain intact requires slower, explicit visual and runtime checks.
Bank-provider choiceCSV/staged import is already an engineering stream. Direct aggregation adds vendor, OAuth, privacy, token, webhook, and support scope.
Assistant authorityRead-only Q&A is a modest feature. Transactional chat is a security, idempotency, audit, recovery, and product-policy program.
Product-scope changesNew roadmap requests are welcome, but each should be placed behind the active phase gate unless it fixes an observed blocker.
Formal verification depthA supervised learning cohort and a broadly available beta are different thresholds; the cohort never substitutes for the 10/10 release system.
