Help & how it works

CashCompass — guiding your money decisions. Documentation for the web dashboard; main areas match the nav below the header (Overview, Assets, Cash Flow, Activity, Properties, Planning). The same product mark appears in the app top bar.

Introduction

CashCompass reads and updates your Google Sheets (INPUT and SYS tabs). Use Run Planner + Refresh Snapshot to sync latest values into the overview and run the debt planner.

Numbers come from your spreadsheets; this UI does not store its own copy of balances.

Overview

Snapshot cards show totals for cash, investments, real estate, debt, and net worth. Some cards include change vs prior month (or week, where noted)—compares live sheet totals to a prior period from INPUT history or OUT - History as documented in your project notes.

Click a card to jump to the related workspace tab when that shortcut exists.

Suggested Actions lists proactive nudges (cash flow, buffer runway, retirement, etc.). Issues is the full list of detected problems (utilization, buffers, upcoming overdue, …). They are separate so items are not duplicated. A short line under Suggested Actions reminds you to scroll to Issues when alerts exist — it does not repeat the issue count.

Assets

House Values — Edits INPUT - House Values for the selected month. SYS - House Assets Current Value is refreshed from the latest values in the current year block when you run the planner (other columns on SYS, such as Type and Loan Amount Left, are not overwritten by that sync).

Bank Accounts — Update saves balances for the selected account. Add new appends one row to INPUT - Bank Accounts (current calendar year block) and one to SYS - Accounts; the workbook must already have that Year block. If you use an opening balance, its date must fall in that same year. Type and Use policy can follow suggestions from existing accounts or be new values.

Add new logs bank_account_add on LOG - Activity and does not run the full planner—use Run Planner + Refresh Snapshot when you want projections and the overview updated.

Investments — Update INPUT sheets; SYS tabs reflect synced balances.

Cash Flow

Quick add — Adds an expense or income line to INPUT - Cash Flow for the selected year. The info box can show the previous month’s value for the same payee: it reads the prior calendar month’s cell on the correct year’s Cash Flow tab (January uses the previous year’s tab for December).

Upcoming, Donations, and Bills Due are separate tabs on this page—see Upcoming expenses, Donations, and Bills Due.

Upcoming expenses

Data lives on INPUT - Upcoming Expenses. The Upcoming tab shows mini-cards (Next 7 Days, Next 30 Days, Overdue, Total Planned) and a list of planned items with amount, due date, payee, category, and source.

Add Upcoming Expense — Saves a new row on that sheet. Auto Add To Cash Flow (checkbox) tells the backend to post to Cash Flow when you save, per your script rules.

List actions — Planned / Paid / Skipped update status. Add to Cash Flow appears when the item is not skipped and not already marked added to Cash Flow. Open in Quick add pre-fills the Quick add form so you can edit before saving.

Status messages — Save and list actions report under the form in Upcoming. After a successful Quick add save, the debt planner may run in the background; if that step fails, the error appears in Quick add’s status line just below the payment form, not next to Run Planner.

Activity log — Saving an upcoming item, changing status (Planned / Paid / Skipped), or pushing to Cash Flow from this tab writes LOG - Activity rows (upcoming_add, upcoming_status, upcoming_cashflow). Pushing from Upcoming uses the same Cash Flow write as Quick add but logs upcoming_cashflow instead of a duplicate quick_pay line. Opening Quick add from an upcoming row and saving there still records quick_pay like any manual Quick add.

Donations

Cash Flow → Donations appends one row to INPUT - Donation for the tax-year section you pick. It does not write to INPUT - Cash Flow—only your charitable-giving log for records and taxes.

Where rows go — New entries are written under the matching Year block: after the row with Year in column A and the year in column B, the next row must be the header Name of Charity, Date, Amount, Tax Year, Comments, Payment type (labels exactly). Data rows follow until the next Year row or the end of the sheet. The script adds your line after the last non-empty row in that block.

Row formatting — When the block already has at least one data row, the new row copies number formats, fonts, borders, and fills from that last existing row (format only; your typed values stay). The first row in a new empty block still gets default date and currency formats on Date and Amount.

Recent donations — The right-hand list shows the newest rows found in all year blocks (by date when the sheet has real dates), similar in spirit to recent house expenses.

Tax year dropdown — Lists years found from Year rows. If a year is missing from the sheet, add that block on INPUT - Donation before using the UI.

Amount — Negatives are allowed (e.g. adjustments). Currency-style typing works like elsewhere in the dashboard.

Name of Charity — Dropdown from existing names on the sheet; Other… opens a field for a new name.

Payment type — Required. Values like Check #1234 on the sheet are grouped as a single Check choice; when you pick Check, enter the check number and the row is saved as Check # plus that number. Use Other… for anything else (e.g. a one-off label).

Activity log

The Activity top-nav page shows actions this script has recorded—not the same thing as OUT - History. OUT - History stores planner run snapshots (rolled-up cash, debt, net worth, etc.). LOG - Activity stores discrete events (who was paid, when it was logged, skip/autopay, etc.) and is created automatically if missing.

What gets logged today

  • Quick add — When you save into INPUT - Cash Flow from the Cash Flow quick-add flow (event type quick_pay on the sheet; optional matching debt balance update for expense payees tied to INPUT - Debts).
  • Bill skip — When Skip writes 0 into Cash Flow for that bill cycle (not when the cell already had a value).
  • Bill autopay — When INPUT - Bills autopay writes the default amount for a due, eligible bill (deduplicated so dashboard refreshes don’t create duplicate rows).
  • House expense — When you save Add House Expense (append to the property’s HOUSES - … sheet). If you also post to INPUT - Cash Flow, the script logs house_expense and does not add a separate quick_pay row for the same save.
  • Donation — When you save Add donation on Cash Flow → Donations (append to INPUT - Donation). Event type on the sheet is donation; the Activity Type column shows Donation.
  • Upcoming expenses — upcoming_add when you save a new row on INPUT - Upcoming Expenses; upcoming_status when you set Planned / Skipped or mark Paid (includes whether Cash Flow was written in that Paid action); upcoming_cashflow when Add to Cash Flow or Paid posts the amount to INPUT - Cash Flow (details include the same previous/new cell snapshot as Quick add, plus the upcoming row id).
  • New bank account — bank_account_add when you create an account from Assets → Bank Accounts → Add new (same event from the sidebar HTML if you use that entry point).

Remove — On the dashboard, Remove is enabled only for Donation log rows until safe undo exists for other event types. For those donations: after confirmation, the log line is removed; if INPUT - Donation still matches (charity, date, amount, tax year, payment type, comments), that sheet row is removed too. For quick_pay (Quick add), bill skip, house expense, etc., the button is greyed out—delete those lines on the LOG - Activity sheet if needed, or wait for a future release.

Filters — Date range applies to when the row was logged (server timestamp). Payee filters rows whose payee contains your text (case-insensitive). Type lists every activity type that appears anywhere in LOG - Activity (derived the same way as the Type column); choose one to limit results. Amount min / max are optional and filter on the logged amount (you can type currency-style numbers). Up to 500 matching rows load per Apply; the table shows 20 at a time with Previous / Next when there are more.

Table columns

  • Logged at — When the script appended the row (shown in your browser’s locale).
  • Payee — Cash Flow / bill payee for the action.
  • Type — A short label for scanning: Loan, Bill, HOA, Tuition, Income, Donation, Upcoming (for upcoming_add / upcoming_status / upcoming_cashflow), Bank (for bank_account_add), Repair / Maintenance / Utilities / Property Tax / etc. (the Type you pick on House Expenses), or House Expenses if that field was blank, plus Other. It is derived for the UI: we match the payee to INPUT - Debts (Account Name + Type) and INPUT - Bills (Payee + Category), keyword checks for HOA and tuition, donation events map to Donation, and for house_expense events we use the log Category column (same as the House Expenses form type). It is not a separate column you edit on LOG - Activity.
  • Amount — The amount stored for that event (e.g. payment amount, or 0 for a skip).
  • Bill due date — The obligation date for that cycle (from the log’s entry date), not necessarily the same calendar day you clicked the button.

Sorting — Click a column header to sort; click again to flip ascending/descending. Sorting applies to all rows that matched your filters on the last Apply (up to 500), then the table shows a page of 20 — not only the current page slice.

Source of truth — Cash Flow, Debts, and other INPUT/SYS tabs remain authoritative for balances. The Activity page helps you audit what the automation recorded; use the sheet LOG - Activity if you want the raw grid or to copy rows.

Bills Due (Cash Flow tab)

Three groups: Overdue, Next 7 Days, and Recurring Bills (No Due Date). Each card shows only the bill name, amount, and due date (where applicable), plus Pay and Skip.

Pay opens Quick add with payee, due date, and amount pre-filled. You can change the amount there before saving—there is no separate “suggested” line on the card; the amount shown is the value the dashboard is using (same number Pay starts with).

Where amounts come from — Rows from INPUT - Bills use the bill fields from that sheet. Items tied to INPUT - Debts (minimum payments) use the planner/debt amount; the card does not list category, autopay, or “varies” flags—those stay on the sheet for your reference.

Default Amount $0 — Bills can use 0 for variable or “enter when known” expenses; they still appear in Overdue / Next 7 Days when the Cash Flow cell for that cycle is empty. Pay opens Quick add with 0—set the real amount there. Autopay never writes a payment when the default amount is 0 (even if Autopay is Yes and Varies is No).

Recurring (no due date) — Bills that are not yet tied to a dated row in INPUT - Bills appear in the third section. Pay opens Quick add with today’s date and amount 0 so you can enter the real payment; map or add due dates in INPUT - Bills when you want them in the dated lists.

Skip marks the bill skipped in the dashboard flow (per your backend rules) and refreshes the lists. Feedback from Pay / Skip (for example “Bill skipped… wrote 0 into Cash Flow”) appears in the status line under the Bills Due heading, not next to Run Planner.

Load errors — If the bills queue cannot be loaded (server or sheet error), the message appears under the Bills Due heading and in the Overview Bills Due card summary. It does not use the top Run Planner status line.

Properties

House Expenses — Adds rows to the HOUSES - … tabs (per property).

Property performance — See the next section for how rent, expenses, and the Type column on SYS - House Assets work together.

Property performance (detail)

Rent — The table sums Income rows on INPUT - Cash Flow for the selected year where the Payee starts with Rent plus the same house name as in SYS - House Assets (for example Rent Oakley House - …).

Rent only for rentals — On SYS - House Assets, the Type column controls this: values Rental (and the common typo Renal) use Cash Flow rent. Types such as Home or Vacation Home show $0 rent in the table even if Income rows exist. If Type is blank, rent is still summed (legacy behavior).

Expenses — These come from each property’s HOUSES - … sheet. The table sums Cost + Service Fees for the selected year from the sheet whose location matches that house (matching is case- and spacing-insensitive).

HOUSES sheet — There must be a HOUSES - … tab that lines up with the House name on SYS for those expenses to show. If there is no matching HOUSES sheet—or the tab name doesn’t match the SYS house (for example SD Condo vs San Diego House)—expense totals stay $0 until you add or rename the HOUSES sheet so it aligns.

Net — Rent minus expenses for that row; the portfolio mini-cards sum the same figures across all houses.

Planning

Debts — Update INPUT - Debts; planner output uses balances and minimums.

Retirement and Purchase Sim — Use the latest snapshot and your inputs; details depend on your sheet configuration.

Payoff Path and Rolling Debt Payoff — Payoff Path is read-only payoff math from the sheet; Rolling Debt Payoff is a separate conservative rolling plan (see its help section).

Payoff Path

Read-only from your sheets — Balances and minimums come from INPUT - Debts. CF paid sums Expense rows on INPUT - Cash Flow for the last two calendar years, where the Payee matches each debt.

Estimated payoff — Loan and HELOC use a monthly amortization loop (APR and minimum from the sheet: interest on the remaining balance, then principal). Other debt types use balance ÷ minimum payment (rough). If the minimum payment is not greater than monthly interest on a loan, payoff is shown as unavailable.

Summary and recommendations on the Payoff Path tab use SYS - Accounts cash and this month’s Cash Flow projection. Simulation / what-if is not part of this view yet.

Rolling Debt Payoff

Separate from Payoff Path — This tab runs a conservative, month-anchored household engine: anchor month is the latest month column on INPUT - Cash Flow that has any Income/Expense data (not today’s calendar date).

Cash out vs minimums — Required cash out uses INPUT - Cash Flow expense lines. INPUT - Debts minimum payments appear as reference only in triggers and JSON; they are not added again on top of Cash Flow (no double count with mortgage or card payments already in CF).

Liquidity model (calculated from SYS - Accounts) — The top KPI row and the Deployable max are derived directly from the accounts sheet; the legacy $100k/$100k hardcoded reserve and buffer constants are retained for audit only and do not drive the numbers you see.

  • Total cash — full liquid total from SYS - Accounts (all cash + checking + savings).
  • Reserve — sum of current balances for accounts flagged DO_NOT_TOUCH.
  • Buffer — sum of the per-account Min Buffer for non-DO_NOT_TOUCH policy-eligible accounts.
  • Cash available to use — Total cash − Reserve − Buffer, before subtracting near-term and unmapped holds.
  • Near-term planned hold — cash reserved for upcoming planned expenses (INPUT - Upcoming Expenses within 90 days).
  • Unmapped card risk hold — temporary safety hold for card-funded planned expenses that are not yet mapped.
  • Deployable max = Total cash − Reserve − Buffer − Near-term hold − Unmapped card risk hold (floored at 0).
  • Cash to use now — your planning input; what you want to deploy this month.
  • Execute now = min(Cash to use now, Deployable max); this is the value the waterfall actually uses. The legacy $50k monthly cap has been retired — your input is the limit.
  • Future / Conditional Cash (not available yet) — variable-income–contingent allocations surfaced below Execution totals. These are not part of Execute now until income arrives.

Operator and Aggressive modes expose the full Cash bridge (audit) and Allocation audit (month 0) panels if you need to trace every deduction step by step.

Income split — Stable income is modeled as Cisco (payee contains “Cisco”) plus the fixed recurring rent payees listed in product rules. Other income is treated as variable; variable inflows use the 50% / 30% / 20% split (debt / reserve+cash / flexible) in the engine.

Forward months — After the anchor month, stable income uses trailing medians. Forward expenses use a recurring operating baseline only: expense payees are bucketed (recurring operating vs debt cleanup vs irregular), and months 1+ sum trailing medians (6→3→1 non-empty fallback per payee) for recurring_operating payees only — not total cash-out. Credit-card payees with high volatility vs their own median go to debt cleanup; taxes, repairs, and similar keywords go to irregular. Variable income in projections defaults to $0 (conservative).

Debt ordering — Extra principal (after the reserve rule) goes to credit cards by highest APR first, then HELOC, Tahoe, San Diego, Missouri, and San Jose last. Loan Depot - San Jose House and any debt with APR ≤ 2.25% get LOW_RATE_KEEP_LAST on the normalized debt row.

Confidence — overall_plan_confidence is a simple rubric (see JSON overall_plan_confidence_rubric); credit-card “spend” without statement balance tracking is marked LOW.

Operating vs forecast — The dashboard leads with THIS MONTH PLAN, an action decision box, and a next-three-month preview. The 12-month cash table and yearly rows are forecasts only (collapsed by default), not a month-by-month instruction list.

Plan status — OK when the core forward model is clean. PARTIALLY_VALID only when an execution-integrity check fails (cash split vs execution total, missing per-account lines when cash extra is non-zero, or total executable mismatch). Advisory noise (expense confidence, irregular payees, unmapped planned card modeling) does not change status. NEEDS_REVIEW when sheet triggers fire; INVALID when income/expense forward numbers cannot be trusted. Turn on Include DEBUG DETAILS on the Rolling Debt Payoff tab for full tables, diagnostics, and JSON.

JSON — Includes this_month_plan, action_decision_box, next_3_month_preview, plus summary, key_warnings, next_12_months, yearly_projection (with debt_reduction_vs_prior_year = prior year ending debt minus this year ending), debt_priority, property_watchlist, assumptions, confidence, and diagnostics (including expense_baseline irregular flags). Anchor month in summary uses yyyy-MM.

Debt Planner email

When you run the debt planner, you may get a short email titled Debt Planner Update. It is meant as a quick glance: bill buckets first, then a compact summary (stability, net worth, cash flow, extra-payoff hints). Full definitions live here—not in the email.

Overdue bills — Same idea as Bills Due → Overdue: due date is before today and the matching INPUT - Cash Flow month cell for that cycle does not yet count as having a value (handled). Includes items tied to INPUT - Debts and INPUT - Bills.

Pay now — Scheduled amounts in the next 7 days (from the day the planner runs): INPUT - Debts minimums (same “handled this month” rule as below) plus dated rows from INPUT - Bills whose Cash Flow cell for that cycle is still empty. If the same payee and due date appears as both a debt minimum and a bill, the debt line is used once (no double count).

Pay soon — Same combined sources, for due dates from day 8 through day 30 out.

Bills Due page — Uses the same underlying queues (bills + debt-matched expenses). Next 7 Days on the web is calendar-based through seven days; Pay now in this email uses the same 7-day window from the run date, so counts usually line up.

Already paid this month — If the expense row for that debt already has a value in the current month column on Cash Flow (same “handled” rule as Bills Due), that debt is omitted from Pay now / Pay soon for that run. Re-run the planner after updating Cash Flow to refresh the email.

Recommended total to pay now — Appears in the email only when the planner suggests an extra payment on top of Pay now minimums. If there is no extra, the Pay now total is all you need for minimums; the summary section still notes the best card target and liquidity when relevant.

Sheet names (quick reference)

  • INPUT - House Values — Year blocks, monthly valuations per house.
  • SYS - House Assets — One row per property: House, Type, Loan Amount Left, Current Value (and sync targets for value).
  • HOUSES - {House} — Expense detail per property; name should match SYS for expense rollups.
  • INPUT - Cash Flow {year} — Monthly columns; Rent payees for rental income.
  • INPUT - Bills — Dated bills (category, autopay, varies, etc.); drives most Bills Due cards.
  • INPUT - Upcoming Expenses — Planned expenses; edited from Cash Flow → Upcoming.
  • INPUT - Debts — Loan accounts; minimum payments can appear on Bills Due.
  • INPUT - Donation — Charitable giving log by tax-year blocks; add rows from Cash Flow → Donations.
  • LOG - Activity — Audit log (Quick add / quick_pay, bill skip, INPUT - Bills autopay, house expense, donations); filter, sort, paginate, and remove mistaken rows on the Activity page.
