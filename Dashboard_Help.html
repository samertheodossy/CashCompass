Help & how it works

CashCompass — guiding your money decisions. This page documents the web dashboard. The top-nav areas (Overview, Assets, Cash Flow, Activity, Properties, Planning) match the sections below.

Introduction

CashCompass reads and updates your Google Sheets (the INPUT - and SYS - tabs). The UI does not keep its own copy of balances — numbers always come from the sheets.

Use Run Planner + Refresh Snapshot to sync the latest sheet values into Overview and re-run the debt planner.

This page is organized in the same order as the top nav, followed by the two debt views (Debt Overview, Rolling Debt Payoff), the planner email, and a sheet-name quick reference.

Overview

Snapshot cards summarize totals for cash, investments, real estate, debt, and net worth.

  • Change vs prior period — Some cards show change vs prior month (or week, where noted). The comparison uses INPUT history or OUT - History.
  • Shortcuts — Clicking a card jumps to the related workspace tab when a shortcut is available.
  • Suggested Actions — Proactive nudges (cash flow, buffer runway, retirement, etc.).
  • Issues — Full list of detected problems (utilization, buffers, upcoming overdue, …). Kept separate from Suggested Actions so items are not duplicated.

Note: A short line under Suggested Actions reminds you to scroll to Issues when alerts exist; it does not repeat the issue count.

Assets

House Values

  • Update saves the valuation for the selected house on the selected month. Edits go to INPUT - House Values.
  • Add new appends one row to INPUT - House Values (current calendar-year block), one row to SYS - House Assets, and creates a new HOUSES - {House} sheet with the canonical expense headers (Item, Type, Date, Location, Cost, Service Fees Paid, Insurance covered, Payments Links, Notes). The workbook must already have that Year block on INPUT - House Values.
  • Required Add-new fields: House name, Property type, Current value, Loan amount left. Valuation date is optional and must fall in the current year when provided — the Current value is seeded into that month. Property type can be chosen from existing houses or typed as a new value.
  • Loan Amount Left is written into column 2 of the new INPUT - House Values row; the matching SYS - House Assets row is created with the same Loan Amount Left and Current Value. Row formatting and column widths are copied from an existing HOUSES sheet so the new sheet visually matches.
  • Duplicate names are rejected across INPUT - House Values, SYS - House Assets, and existing HOUSES - * tab names. Inline errors show next to the House name field and focus the input.
  • Add new logs house_add to LOG - Activity. It does not re-run the full planner — use Run Planner + Refresh Snapshot when you want projections and Overview updated.
  • After a successful add, the House Values dropdown re-selects the new house, and the Properties → House Expenses selector refreshes so the house is immediately available there.
  • Running the planner refreshes SYS - House Assets → Current Value from the latest values in the current-year block. Other SYS columns (Type, Loan Amount Left) are not overwritten by that sync.

Bank Accounts

  • Update saves the balance for the selected account.
  • Add new appends one row to INPUT - Bank Accounts (current calendar-year block) and one to SYS - Accounts. The workbook must already have that Year block.
  • If you supply an opening balance, its date must fall in that same year.
  • Type and Use policy can be chosen from existing accounts or typed as new values.
  • Add new logs bank_account_add to LOG - Activity. It does not re-run the full planner — use Run Planner + Refresh Snapshot when you want projections and Overview updated.

Investments

  • Edits go to the relevant INPUT sheets. SYS tabs reflect synced balances.

Cash Flow

The Cash Flow page is split into several tabs. The core of the page is Quick add; the other tabs (Upcoming, Donations, Bills) have their own help sections below.

Quick add

  • Adds an Expense or Income line to INPUT - Cash Flow for the selected year.
  • The preview box shows the same payee’s previous-month value when available. It reads the prior calendar month on the correct year’s Cash Flow tab — January uses the previous year’s tab for December.

Upcoming expenses

What it does

  • Reads and writes INPUT - Upcoming Expenses.
  • The tab shows mini-cards (Next 7 Days, Next 30 Days, Overdue, Total Planned) and a list of planned items with amount, due date, payee, category, and source.

Actions

  • Add Upcoming Expense — Saves a new row. The optional Auto Add To Cash Flow checkbox tells the backend to post the row to Cash Flow on save (per your script rules).
  • Planned / Paid / Skipped — Updates the row’s status.
  • Add to Cash Flow — Appears when the item is not skipped and not already marked added. Writes to Cash Flow directly.
  • Open in Quick add — Pre-fills the Quick add form so you can edit before saving.

Status messages

  • Save and list actions report under the form on the Upcoming tab.
  • After a successful Quick add save, the debt planner may run in the background. If that step fails, the error appears in Quick add’s status line — not next to Run Planner.

Activity log

  • upcoming_add — new row saved on this tab.
  • upcoming_status — status changed (Planned / Paid / Skipped).
  • upcoming_cashflow — row pushed to Cash Flow via Add to Cash Flow or Paid.

Notes: Pushing from Upcoming uses the same Cash Flow write as Quick add but logs upcoming_cashflow instead of a duplicate quick_pay. Opening Quick add from an upcoming row and saving there still records quick_pay like any manual Quick add.

Donations

What it does

  • Appends one row to INPUT - Donation under the tax-year block you pick.
  • Does not write to INPUT - Cash Flow — this is the charitable-giving log for records and taxes.

Where rows go

  • Year blocks start with a row where column A is Year and column B is the year.
  • The next row must be the literal header row: Name of Charity, Date, Amount, Tax Year, Comments, Payment type.
  • Data rows follow until the next Year row or the end of the sheet.
  • The script inserts after the last non-empty row in the matching block.

Row formatting

  • When the block already has data, the new row copies number formats, fonts, borders, and fills from the last existing row (format only; your typed values are preserved).
  • The first row in a brand-new block still gets default date and currency formats on Date and Amount.

Form fields

  • Tax year — Dropdown built from existing Year rows. Add the block on the sheet first if it’s missing.
  • Name of Charity — Dropdown from existing names. Pick Other… to type a new one.
  • Amount — Negatives are allowed (e.g. adjustments). Currency-style typing works like elsewhere.
  • Payment type — Required. Values like Check #1234 collapse into a single Check choice; picking Check prompts for a number and the row is saved as Check # + number. Use Other… for one-off labels.

Recent donations

  • The right-hand list shows the newest rows from all year blocks (sorted by date when the sheet has real dates), similar to recent house expenses.

Activity log

What it does

The Activity page shows events this script has recorded. It is not the same as OUT - History.

  • OUT - History — Snapshots from planner runs (rolled-up cash, debt, net worth, etc.).
  • LOG - Activity — Discrete events (who was paid, when it was logged, skip/autopay, etc.). Created automatically if missing.

What gets logged — Cash Flow actions

  • quick_pay — Save in Quick add. For expense payees tied to INPUT - Debts, the matching debt balance is optionally updated.
  • house_expense — Save in Add House Expense (see Properties). If the same save also posts to Cash Flow, only house_expense is logged — no duplicate quick_pay.

What gets logged — Bills actions

  • bill_skip — Skip writes 0 into Cash Flow for that cycle. Not logged if the cell already had a value.
  • bill_autopay — Autopay writes the default amount for a due, eligible bill. Deduplicated so dashboard refreshes do not create duplicates.
  • bill_add — New recurring bill saved via Cash Flow → Bills → Add bill. Appended to INPUT - Bills. Activity Type is Bill; Action sub-label is Bill added.
  • bill_deactivate — Stop tracking on a recurring bill (sets Active = No; the row is kept for history, no payments are reversed). Activity Type is Bill; sub-label is Tracking stopped; Amount shows — because no money moved.

What gets logged — Upcoming, Donations, Property, Bank

  • upcoming_add / upcoming_status / upcoming_cashflow — See Upcoming expenses.
  • donation — Save on Cash Flow → Donations. Activity Type shows Donation.
  • house_expense — Save on Add House Expense, covered above.
  • bank_account_add — Create an account from Assets → Bank Accounts → Add new (same event from the sidebar HTML).
  • house_add — Create a house from Assets → House Values → Add new. Adds rows to INPUT - House Values and SYS - House Assets and creates the matching HOUSES - {House} sheet. Activity Type is House Expenses; Action sub-label is House added. Amount shows the house's initial Current Value.

Remove button

  • Enabled only for Donation rows until safe undo exists for other event types.
  • For donations: after confirmation, the log line is removed; if INPUT - Donation still has a matching row (charity, date, amount, tax year, payment type, comments), that sheet row is removed too.
  • For other events (quick_pay, bill skip, house expense, bank account add, house_add, …) the button is greyed out — delete those lines directly on LOG - Activity if needed.

Filters

  • Date range — Applies to when the row was logged (server timestamp).
  • Payee — Substring match, case-insensitive.
  • Type — Lists every activity type present anywhere in LOG - Activity, derived the same way as the Type column.
  • Amount min / max — Optional currency-style inputs; filter on the logged amount.
  • Up to 500 matching rows load per Apply; the table shows 20 at a time with Previous / Next.

Table columns

  • Logged at — When the script appended the row (browser locale).
  • Payee — The Cash Flow / bill payee for that action.
  • Type — A short label derived for scanning: Loan, Bill, HOA, Tuition, Income, Donation, Upcoming, Bank, Repair / Maintenance / Utilities / Property Tax (from the House Expenses Type), House Expenses if that field was blank, or Other.
    • It is derived, not a column you edit. Payees are matched to INPUT - Debts (Account Name + Type) and INPUT - Bills (Payee + Category); keyword checks cover HOA and tuition; donation events map to Donation; house_expense events use the log Category.
    • Under the pill, a small Action sub-label disambiguates events that share a Type (e.g. the Bill pill can show Bill added, Tracking stopped, Bill skipped, or Bill autopay).
    • The Type filter operates on the broad pill, so filtering by Bill still surfaces every bill lifecycle event.
  • Amount — The amount stored for the event (payment amount, or 0 for a skip). Lifecycle events that do not move money — Tracking stopped (bill_deactivate) and Bank account creation — show — instead of $0.00.
  • Bill due date — The obligation date for that cycle (from the log entry date). Not necessarily the day you clicked the button.

Sorting

  • Click a column header to sort; click again to flip ascending/descending.
  • Sorting applies to all rows that matched your filters on the last Apply (up to 500) — not only the current page.

Source of truth

Cash Flow, Debts, and the other INPUT/SYS tabs remain authoritative for balances. The Activity page helps you audit what the automation recorded; open LOG - Activity directly for the raw grid or to copy rows.

Bills

The Bills page (on Cash Flow) has two internal views: Due this period for execution, and Manage bills for the underlying list on INPUT - Bills.

Due this period

  • Three groups: Overdue, Next 7 Days, and Recurring Bills (No Due Date).
  • Each card shows the bill name, amount, and due date (when applicable), plus Pay and Skip.
  • The card does not list category, autopay, or “varies” flags — those remain on the sheet for your reference.

What the buttons do

  • Pay — Opens Quick add with payee, due date, and amount pre-filled. You can change the amount before saving. The amount shown on the card is the same number Pay starts with; there is no separate “suggested” line.
  • Skip — Marks the bill skipped (per your backend rules) and refreshes the lists. Typically writes 0 into the matching Cash Flow cell.

Where amounts come from

  • Rows from INPUT - Bills use that sheet’s bill fields.
  • Items tied to INPUT - Debts (minimum payments) use the planner / debt amount.

Default Amount = $0

  • Bills can use 0 for variable or “enter when known” expenses. They still appear in Overdue / Next 7 Days when the Cash Flow cell for that cycle is empty.
  • Pay opens Quick add with 0 — set the real amount there.
  • Autopay never writes a payment when the default amount is 0 (even if Autopay is Yes and Varies is No).

Recurring (no due date)

  • Bills not yet tied to a dated row in INPUT - Bills land in the third section.
  • Pay opens Quick add with today’s date and amount 0 so you can enter the real payment.
  • Map or add due dates on INPUT - Bills when you want them in the dated lists.

Manage bills

  • Lists every row from INPUT - Bills with sortable columns (Payee, Due Day).
  • Default sort is Due Day, ascending. Click a column header to sort; click again to flip direction.

Add bill

  • Appends a row to INPUT - Bills. Category is required (both client and server side).
  • Logs bill_add on LOG - Activity.

Stop tracking

  • Sets Active = No on the bill row. The row is kept for history; no payments are reversed.
  • Logs bill_deactivate (sub-label Tracking stopped, amount —).

Status & errors

  • Feedback from Pay / Skip and from Add bill / Stop tracking appears in the status line under the Bills heading, not next to Run Planner. Success messages auto-clear.
  • If the bills queue fails to load, the error shows under the Bills heading and in the Overview Bills Due card summary.

Properties

  • House Expenses — Adds rows to the per-property HOUSES - … tabs. The house selector refreshes every time you open the tab, so houses created via Assets → House Values → Add new appear immediately (their HOUSES - {House} sheet is created automatically at add time).
  • Property performance — See the next section for rent, expenses, and how the Type column on SYS - House Assets ties them together.

Property performance (detail)

Rent

  • Sums Income rows on INPUT - Cash Flow for the selected year.
  • Only counts rows whose Payee starts with Rent followed by the same house name as in SYS - House Assets — for example, Rent Oakley House - ….

Rent only for rentals

  • The Type column on SYS - House Assets controls this.
  • Rental (and the common typo Renal) use Cash Flow rent.
  • Home and Vacation Home show $0 rent even if Income rows exist.
  • A blank Type still sums rent (legacy behavior).

Expenses

  • Come from each property’s HOUSES - … sheet.
  • The table sums Cost + Service Fees for the selected year on the sheet whose location matches the house (matching is case- and spacing-insensitive).

HOUSES sheet

  • There must be a HOUSES - … tab aligned with the SYS House name. Assets → House Values → Add new creates this sheet automatically with the canonical header layout and column widths copied from an existing HOUSES tab.
  • If no matching HOUSES sheet exists — or the name differs (e.g. SD Condo vs San Diego House) — expense totals stay $0 until you add or rename the tab.

Net

  • Net = Rent − Expenses for that row.
  • The portfolio mini-cards sum the same figures across all houses.

Planning

  • Debts — Edit INPUT - Debts; planner output uses the balances and minimums there.
  • Retirement and Purchase Sim — Use the latest snapshot plus your inputs. Details depend on sheet configuration.
  • Debt Overview vs Rolling Debt Payoff:
    • Debt Overview — Read-only reference view of your debt structure.
    • Rolling Debt Payoff — The monthly action decision engine. See its own section below.

Debt Overview

What it is

  • Reference view, not an action planner. Shows balances, minimums, APRs, and estimated payoff at current minimums.
  • For monthly action decisions, use the Rolling Debt Planner instead.

Data sources

  • Balances and minimums — From INPUT - Debts.
  • CF paid — Sums Expense rows on INPUT - Cash Flow for the last two calendar years where the Payee matches the debt.

Estimated payoff

  • Loan and HELOC — Monthly amortization loop using the sheet’s APR and minimum: interest on remaining balance, then principal.
  • Other debt types — Rough estimate as balance ÷ minimum payment.
  • If the minimum is not greater than monthly interest on a loan, payoff is shown as unavailable.

Summary and notes

The page’s summary/notes use SYS - Accounts cash and this month’s Cash Flow projection as descriptive context. Simulation / what-if is not part of this view.

Rolling Debt Payoff

This is the monthly decision engine — separate from Debt Overview (which is a read-only reference). The anchor month is the latest month column on INPUT - Cash Flow that has any Income/Expense data — not today’s calendar date.

Standard view

The tab opens in a compact decision layout. Everything here is driven by the same planner JSON; the Show details toggle at the bottom expands the full forecasts, audits, and diagnostics.

Cash to use now

  • Single input at the top. Type how much you want to deploy this month.
  • The planner caps it at Safe to use (your deployable max after reserves, buffers, and holds) and reruns the waterfall instantly.
  • A short live formula line shows min(Cash to use now, Safe to use).

Decision card

A three-line narrative summary:

  • Recommendation — What the plan suggests (e.g. deploy $X now, focus on Account Y).
  • Why — The short reason (highest-APR target, balance caps, or a strategy choice).
  • Caution — Any near-term risk or reason the recommendation is conservative.

HELOC strategy card

  • Appears when a HELOC is modeled.
  • Shows the recommended action and the draw size.
  • What would change this? lists the actionable levers (HELOC APR, eligible-debt spread, upcoming expense hold, recurring monthly repayment) with current values, so you can see what’s driving the recommendation.

Payment result — what the numbers mean

A compact table of the accounts the plan actually touches this month:

  • Account — The debt name plus its role in the waterfall:
    • Small balance — paid first to close small accounts.
    • Focus debt — the highest-priority target this month.
    • Next debt — spill once Focus debt is capped.
    • Excess — anything else (e.g. a low-rate line kept for last).
  • Action — Paid off (this month) when the plan clears the balance (shown green), or Partially paid when it reduces the balance.
  • Remaining — Projected balance after this month’s payment.
  • [Add payment] — Small pill on rows with a real payable amount. Clicking it jumps to Cash Flow → Quick add pre-filled with the payee and the planner’s amount; review or adjust, then save. Hidden in standalone contexts where Quick add is not available.

Why not more?

  • Shown when the plan deploys less than Safe to use.
  • Explains the two holds that caused it: near-term planned hold and unmapped card risk hold.
  • Each bullet has an inline [View] pill that opens Cash Flow → Upcoming when the hold is driven by planned upcoming expenses, so you can act on the underlying rows.

Strategy control

  • Small toggle to switch between the conservative default and a more aggressive concentration (e.g. push a larger share of the remainder onto Focus debt).
  • The decision card and payment result update in place.

Safe to use (liquidity model)

Derived from SYS - Accounts. The legacy $100k / $100k hardcoded reserve and buffer constants are retained for audit only and no longer drive the numbers.

  • Total cash — Full liquid total from SYS - Accounts (all cash + checking + savings).
  • Reserve — Sum of current balances for accounts flagged DO_NOT_TOUCH.
  • Buffer — Sum of per-account Min Buffer for non-DO_NOT_TOUCH, policy-eligible accounts.
  • Cash available to use — Total cash − Reserve − Buffer, before subtracting holds.
  • Near-term planned hold — Cash reserved for upcoming planned expenses within 90 days. First driver behind Why not more?.
  • Unmapped card risk hold — Safety hold for card-funded planned expenses not yet mapped. Second Why not more? driver; when fed by Upcoming rows, the bullet exposes the same [View] action.
  • Safe to use (Deployable max) = Total cash − Reserve − Buffer − Near-term hold − Unmapped card risk hold (floored at 0).
  • Cash to use now — Your planning input.
  • Execute now = min(Cash to use now, Safe to use). This is the value the waterfall uses. The legacy $50k monthly cap has been retired — your input is the limit.
  • Future / Conditional Cash (not available yet) — Variable-income-contingent allocations surfaced below execution totals. Not part of Execute now until income arrives.

Show details — planner mechanics

Show details expands the full output: cash-bridge audit, allocation audit (month 0), 12-month cash table, yearly projection, forecasts, diagnostics, and raw JSON. Click Hide details to collapse back.

  • Cash out vs minimums — Required cash out uses INPUT - Cash Flow expense lines. INPUT - Debts minimums appear as reference only in triggers and JSON; they are not double-counted on top of Cash Flow.
  • Income split — Stable income = Cisco (payee contains “Cisco”) plus the fixed recurring rent payees in product rules. Other income is variable; variable inflows use the 50% / 30% / 20% split (debt / reserve+cash / flexible).
  • Forward months — After the anchor month, stable income uses trailing medians. Forward expenses use a recurring operating baseline only: expenses are bucketed (recurring operating / debt cleanup / irregular) and future months sum trailing medians (6 → 3 → 1 non-empty fallback per payee) for recurring_operating payees only — not total cash-out. High-volatility credit-card payees go to debt cleanup; taxes, repairs, and similar keywords go to irregular. Variable income in projections defaults to $0 (conservative).
  • Debt ordering — Extra principal (after the reserve rule) goes to credit cards by highest APR first, then HELOC, Tahoe, San Diego, Missouri, and San Jose last. Loan Depot - San Jose House and any debt with APR ≤ 2.25% get LOW_RATE_KEEP_LAST on the normalized debt row.
  • Role names in the waterfall — Presentation labels only; math and JSON field names are unchanged.
    • Small balance → engine’s Cleanup bucket.
    • Focus debt → Primary.
    • Next debt → Secondary.
    • Excess → Overflow.
  • Confidence — overall_plan_confidence is a simple rubric (see JSON overall_plan_confidence_rubric); credit-card “spend” without statement-balance tracking is marked LOW.
  • Operating vs forecast — The Standard view is the operating picture (this month’s decision). The 12-month cash table, yearly rows, and next-three-month preview under Show details are forecasts, not a month-by-month instruction list.
  • Plan status:
    • OK — Core forward model is clean.
    • PARTIALLY_VALID — An execution-integrity check failed (cash split vs execution total, missing per-account lines when cash extra is non-zero, or total executable mismatch). Advisory noise does not change status.
    • NEEDS_REVIEW — Sheet triggers fire.
    • INVALID — Income/expense forward numbers cannot be trusted.
  • JSON highlights — this_month_plan, action_decision_box, next_3_month_preview, summary (anchor month as yyyy-MM), key_warnings, next_12_months, yearly_projection (with debt_reduction_vs_prior_year = prior year ending debt minus this year ending), debt_priority, property_watchlist, assumptions, confidence, diagnostics (including expense_baseline irregular flags).

Debt Planner email

Running the debt planner may send a short email titled Debt Planner Update. It is a quick glance — bill buckets first, then a compact summary (stability, net worth, cash flow, extra-payoff hints). Full definitions live here, not in the email.

What you see

  • Overdue bills — Same idea as Bills → Overdue: due date is before today and the matching Cash Flow month cell for that cycle does not yet count as having a value. Combines INPUT - Debts and INPUT - Bills.
  • Pay now — Scheduled amounts due in the next 7 days from the run date: INPUT - Debts minimums (with the “handled this month” rule below) plus dated rows from INPUT - Bills whose Cash Flow cell for that cycle is still empty. If the same payee + due date appears as both, the debt line is used once — no double count.
  • Pay soon — Same combined sources, for due dates day 8 through day 30 out.
  • Recommended total to pay now — Appears only when the planner suggests an extra payment on top of Pay now minimums. Otherwise, Pay now covers minimums; the summary section still notes the best card target and liquidity when relevant.

“Already paid this month” rule

  • If the expense row for a debt already has a value in the current month column on Cash Flow (same “handled” rule as Bills), that debt is omitted from Pay now / Pay soon for that run.
  • Re-run the planner after updating Cash Flow to refresh the email.

Why it matches the Bills page

The Bills page uses the same underlying queues. Next 7 Days on the web is calendar-based through seven days; Pay now in the email uses the same 7-day window from the run date, so counts usually line up.

Sheet names (quick reference)

INPUT sheets

  • INPUT - House Values — Year blocks with monthly valuations per house.
  • INPUT - Cash Flow {year} — Monthly columns; Rent payees drive rental income.
  • INPUT - Bills — Dated bills (category, autopay, varies, payment source, etc.). Drives most Bills cards.
  • INPUT - Upcoming Expenses — Planned expenses. Edited from Cash Flow → Upcoming.
  • INPUT - Debts — Loan accounts; minimums can surface on the Bills page.
  • INPUT - Donation — Charitable giving log organized by tax-year blocks. Added from Cash Flow → Donations.
  • INPUT - Bank Accounts — Per-year balance blocks. Written by Assets → Bank Accounts.

SYS sheets

  • SYS - House Assets — One row per property: House, Type, Loan Amount Left, Current Value.
  • SYS - Accounts — Liquid accounts with use-policy flags; feeds the Safe-to-use liquidity model.

HOUSES sheets

  • HOUSES - {House} — Expense detail per property. Name should match the SYS house for rollups. Created automatically by Assets → House Values → Add new with canonical headers (Item, Type, Date, Location, Cost, Service Fees Paid, Insurance covered, Payments Links, Notes) and column widths copied from an existing HOUSES sheet.

Output & audit

  • OUT - History — Planner run snapshots (cash, debt, net worth).
  • LOG - Activity — Audit log for quick_pay, bill skip / autopay / add / deactivate, house expense, donation, upcoming actions, bank account add, and house add. Filter, sort, paginate, and remove (Donation-only) mistaken rows from the Activity page.
