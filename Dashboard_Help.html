Introduction

CashCompass reads and updates your Google Sheets (the INPUT - and SYS - tabs). The UI does not keep its own copy of balances — numbers always come from the sheets.

Use Run Planner + Refresh Snapshot to sync the latest sheet values into Overview and re-run the debt planner.

New to the app? Open Setup / Review from the top-right. It is safe to open at any time — the flow is read-only until you explicitly open an editor.

This page is organized in the same order as the top nav, followed by the two debt views (Debt Overview, Rolling Debt Payoff), the planner email, and a sheet-name quick reference.

Setup / Review

Setup / Review is the top-right button in the header. It opens a focused walkthrough of the five core input areas CashCompass reads when it decides what to do next. It is safe to open at any time — the flow is read-only until you explicitly open an editor.

Screen flow

  • Welcome — short intro shown only the first time you open Setup / Review with no data in any of the five core areas. Press Continue to open the status grid, or Back to Dashboard to leave. Once any core area has data, opening Setup / Review goes straight to the status grid.
  • Status grid — one card per required area (Bank Accounts, Debts, Bills, Upcoming Expenses, Income). Each card shows Setup complete or Not set up plus a short summary. A separate Optional row below the grid holds Houses.
  • Completion panel — sits under the grid with a readiness summary (You're ready when all five required areas are set up, otherwise Almost there with a short progress line). Primary action is Go to Next Actions; a secondary See setup summary link opens a per-area breakdown.
  • Detail screens — click any card to open a read-only detail view for that area. The detail view shows what is currently tracked and a primary action to open the matching editor.

Steps covered

  • Bank Accounts — reads the current-year block on Bank Accounts. Opens the Assets → Bank Accounts editor in Setup mode. When empty, the CTA becomes Add first bank account.
  • Debts — reads active debt rows (name, balance, minimum payment). Opens the Planning → Debts editor in Setup mode. When empty, the CTA becomes Add first debt.
  • Bills — reads active recurring bills (payee, amount, due day, frequency). Opens the Cash Flow → Bills editor in Setup mode. When empty, the CTA becomes Add first bill.
  • Upcoming Expenses — reads Planned rows only (Paid / Dismissed rows are treated as history). Opens the Cash Flow → Upcoming editor in Setup mode. When empty, the CTA becomes Add first upcoming expense.
  • Income — derived from the latest Cash Flow year sheet. Groups recurring income (seen in ≥3 months, not an excluded category such as Bonus / RSU / Refund) and shows other detected entries for reference. Opens the Cash Flow → Income tab in Setup mode. There is no separate income source sheet.

Optional

  • Houses — optional property tracking. Appears as an Optional-tagged card in its own row below the required steps. Not part of the Setup / Review completion count, not used for cash decisions, and not required to finish Setup. Opens the Assets → House Values editor in Setup mode. If you have no properties yet, the CTA becomes Add first property.

Setup-mode editor handoff

Opening an editor from Setup enters a focused shell: the main top navigation, page sub-tabs, Setup / Review, and Run Planner + Refresh Snapshot are hidden. A slim Back to Setup bar appears at the top. Press it to return to the matching detail screen. Opening the same editor normally (from the main navigation) is unchanged — the full shell remains.

Safety rules

  • Viewing Setup does not write anywhere, does not touch SYS - sheets, and does not log anything in Activity.
  • Editors launched from Setup use the exact same save logic as the normal path — there is no parallel editor.
  • If the input sheet for Bills or Upcoming Expenses is missing, Setup creates it with the correct headers before opening the editor. The Cash Flow year sheet is not auto-created from Setup — if it is missing, the Income step says so explicitly.

Overview

Snapshot cards summarize totals for cash, investments, real estate, debt, and net worth.

  • Change vs prior period — Some cards show change vs prior month (or week, where noted). The comparison uses INPUT history or OUT - History.
  • Shortcuts — Clicking a card jumps to the related workspace tab when a shortcut is available.
  • Suggested Actions — Proactive nudges (cash flow, buffer runway, retirement, etc.).
  • Issues — Full list of detected problems (utilization, buffers, upcoming overdue, …). Kept separate from Suggested Actions so items are not duplicated.

Note: A short line under Suggested Actions reminds you to scroll to Issues when alerts exist; it does not repeat the issue count.

Assets

House Values

  • Update saves the valuation for the selected house on the selected month. Edits go to INPUT - House Values.
  • Add new appends one row to INPUT - House Values (current calendar-year block), one row to SYS - House Assets, and creates a new HOUSES - {House} sheet with the canonical expense headers (Item, Type, Date, Location, Cost, Service Fees Paid, Insurance covered, Payments Links, Notes). The workbook must already have that Year block on INPUT - House Values.
  • Required Add-new fields: House name, Property type, Current value, Loan amount left. Valuation date is optional and must fall in the current year when provided — the Current value is seeded into that month. Property type can be chosen from existing houses or typed as a new value.
  • Loan Amount Left is written into column 2 of the new INPUT - House Values row; the matching SYS - House Assets row is created with the same Loan Amount Left and Current Value. Row formatting and column widths are copied from an existing HOUSES sheet so the new sheet visually matches.
  • Duplicate names are rejected across INPUT - House Values, SYS - House Assets, and existing HOUSES - * tab names. Inline errors show next to the House name field and focus the input.
  • Add new logs house_add to LOG - Activity. It does not re-run the full planner — use Run Planner + Refresh Snapshot when you want projections and Overview updated.
  • After a successful add, the House Values dropdown re-selects the new house, and the Properties → House Expenses selector refreshes so the house is immediately available there.
  • Running the planner refreshes SYS - House Assets → Current Value from the latest values in the current-year block. Other SYS columns (Type, Loan Amount Left) are not overwritten by that sync.

House Values — Stop tracking (soft deactivate)

  • Update mode shows a secondary Stop tracking button next to Save House Value. Use it when you sell a house or otherwise want to retire it from active views without losing history.
  • Clicking it opens an inline confirmation. On confirm:
    • Every row for that house in INPUT - House Values (across all year blocks) is flipped to Active = No — canonical source of truth.
    • The matching SYS - House Assets row mirrors Active = No.
    • Month values, Loan Amount Left, Current Value, Type, and the HOUSES - {House} sheet are left untouched.
  • Inactive houses drop out of the Update House Value dropdown, the Properties → House Expenses selector, and the Properties → Property performance rows + portfolio cards. History stays in every sheet for audit / reporting.
  • The HOUSES - {House} sheet is not deleted or renamed. You can still open it directly on the workbook to review past expenses.
  • Duplicate-name validation still blocks reuse of a stop-tracked house name when adding a new house (names stay reserved).
  • Logs house_deactivate to LOG - Activity. Activity Type is House Expenses; Action sub-label is Tracking stopped; Amount shows — because no money moved.
  • New Yes / No values written into the Active column inherit the row's text style so they visually match the rest of the row.

Cash Flow

The Cash Flow page is split into several tabs. The core of the page is Quick add; the other tabs (Upcoming, Donations, Bills) have their own help sections below.

Quick add

  • Adds an Expense or Income line to INPUT - Cash Flow for the selected year.
  • The preview box shows the same payee’s previous-month value when available. It reads the prior calendar month on the correct year’s Cash Flow tab — January uses the previous year’s tab for December.

Upcoming expenses

What it does

  • Reads and writes INPUT - Upcoming Expenses.
  • Upcoming is an open-obligations board: each row is a planned expense with a remaining amount. The Amount column is the live remaining balance — partial payments reduce it and fully paid rows drop off the active list automatically.
  • The tab shows mini-cards (Next 7 Days, Next 30 Days, Overdue, Total Planned) and a list of items that are still actionable (Planned with remaining > 0).

Actions

  • Add Upcoming Expense — Saves a new row. Expense Name, Due Date, and Amount are required (Amount must be greater than 0); the other fields are optional. Required-field errors appear inline next to the offending field; the status line below the button carries the action-specific loading / error message.
  • Quick add payment — Sends you to Cash Flow → Quick add pre-filled with this row’s payee, due date, and current remaining amount. This is the single payment path for Upcoming: edit the amount if you paid more or less, then save. Cash Flow records the money movement; Upcoming’s remaining balance is decremented automatically and the row is flipped to Paid when it reaches 0.
  • Dismiss — Removes the row from the active board without recording a payment. Use this for obligations you no longer want to track. The sheet row is preserved with Status = Dismissed for history.

Lifecycle model

  • Status is a read-only label: Planned / Paid / Dismissed (and legacy Skipped rows are treated the same as Dismissed for filtering and sort). The old manual Planned / Paid / Skipped buttons and the direct Add to Cash Flow button have been retired in favor of Quick add + Dismiss.
  • Partial payments: paying less than the remaining amount keeps the row on the active board with the reduced balance. The row’s Added To Cash Flow column is flipped to Yes the first time a payment is applied.
  • Full payment: paying the full remaining amount (or more) clamps remaining to 0, flips Status to Paid, and removes the row from the active board. The sheet row is never deleted.
  • Cash Flow + Activity are the money/history trail — Upcoming does not maintain a parallel payment log.

Status messages

  • Action-specific text: Saving upcoming expense…, Opening Quick Add…, Dismissing….
  • When Quick Add is launched from an Upcoming row, the Quick Add status line reports the Cash Flow save followed by the Upcoming remaining-balance update (e.g. Partial payment applied. Remaining: $123.45 or Upcoming expense fully paid.).

Cross-panel refresh

  • Every Upcoming write (add / dismiss / payment via Quick add) re-pulls the Upcoming list, the Bills Due execution queue, and the Overview snapshot so stale counts don’t linger.
  • Navigating to the Upcoming tab also re-loads the list, so new items added from another flow appear without a full page reload.

Activity log

  • upcoming_add — new row saved on this tab. Amount is the originally planned amount.
  • upcoming_payment — a Quick Add payment was applied to this row. Lifecycle-only; Amount renders as — so the dollars aren’t double-counted against the paired quick_pay row. The action label reads Applied $500.00 (Remaining $250.00) for a partial payment, Applied $500.00 (Fully paid) when the payment closes the item, and falls back to Payment applied for legacy rows without paid-amount details. Amount and remaining are read from the existing details JSON (paidAmount, remainingAfter, fullyPaid).
  • upcoming_status — Dismiss event. Lifecycle-only; Amount renders as —.
  • upcoming_cashflow — legacy event from the retired direct Add to Cash Flow path. Still rendered for historical rows; no new events of this type are written.

Notes: The actual money movement from an Upcoming payment is the quick_pay row logged by the Quick Add save (same as any manual Quick Add). The paired upcoming_payment row is the Upcoming-side audit trail only.

Donations

What it does

  • Appends one row to INPUT - Donation under the tax-year block you pick.
  • Does not write to INPUT - Cash Flow — this is the charitable-giving log for records and taxes.

Where rows go

  • Year blocks start with a row where column A is Year and column B is the year.
  • The next row must be the literal header row: Name of Charity, Date, Amount, Tax Year, Comments, Payment type.
  • Data rows follow until the next Year row or the end of the sheet.
  • The script inserts after the last non-empty row in the matching block.

Row formatting

  • When the block already has data, the new row copies number formats, fonts, borders, and fills from the last existing row (format only; your typed values are preserved).
  • The first row in a brand-new block still gets default date and currency formats on Date and Amount.

Form fields

  • Tax year — Dropdown built from existing Year rows. Add the block on the sheet first if it’s missing.
  • Name of Charity — Dropdown from existing names. Pick Other… to type a new one.
  • Amount — Negatives are allowed (e.g. adjustments). Currency-style typing works like elsewhere.
  • Payment type — Required. Values like Check #1234 collapse into a single Check choice; picking Check prompts for a number and the row is saved as Check # + number. Use Other… for one-off labels.

Recent donations

  • The right-hand list shows the newest rows from all year blocks (sorted by date when the sheet has real dates), similar to recent house expenses.

Activity log

What it does

The Activity page shows events this script has recorded. It is not the same as OUT - History.

  • OUT - History — Snapshots from planner runs (rolled-up cash, debt, net worth, etc.).
  • LOG - Activity — Discrete events (who was paid, when it was logged, skip/autopay, etc.). Created automatically if missing.

What gets logged — Cash Flow actions

  • quick_pay — Save in Quick add. For expense payees tied to INPUT - Debts, the matching debt balance is optionally updated.
  • house_expense — Save in Add House Expense (see Properties). If the same save also posts to Cash Flow, only house_expense is logged — no duplicate quick_pay.

What gets logged — Bills actions

  • bill_skip — Skip writes 0 into Cash Flow for that cycle. Not logged if the cell already had a value.
  • bill_autopay — Autopay writes the default amount for a due, eligible bill. Deduplicated so dashboard refreshes do not create duplicates.
  • bill_add — New recurring bill saved via Cash Flow → Bills → Add bill. Appended to INPUT - Bills. Activity Type is Bill; Action sub-label is Bill added.
  • bill_deactivate — Stop tracking on a recurring bill (sets Active = No; the row is kept for history, no payments are reversed). Activity Type is Bill; sub-label is Tracking stopped; Amount shows — because no money moved.

What gets logged — Upcoming, Donations, Property, Bank

  • upcoming_add / upcoming_payment / upcoming_status (Dismiss) / legacy upcoming_cashflow — See Upcoming expenses.
  • donation — Save on Cash Flow → Donations. Activity Type shows Donation.
  • house_expense — Save on Add House Expense, covered above.
  • bank_account_add — Create an account from Assets → Bank Accounts → Add new (same event from the sidebar HTML). Activity Type is Bank; Action sub-label is Account added. Amount shows the supplied opening balance when one was provided (e.g. $150.00), or $0.00 when the account was added without an opening amount.
  • bank_account_deactivate — Stop tracking on a bank account from Assets → Bank Accounts → Update. Flips Active = No on every INPUT - Bank Accounts row for that account plus the mirror SYS - Accounts row; month history, Available Now, and Min Buffer stay in place. Activity Type is Bank; sub-label is Tracking stopped; Amount shows —.
  • house_add — Create a house from Assets → House Values → Add new. Adds rows to INPUT - House Values and SYS - House Assets and creates the matching HOUSES - {House} sheet. Activity Type is House Expenses; Action sub-label is House added. Amount shows the house's initial Current Value.
  • house_deactivate — Stop tracking on a house from Assets → House Values → Update. Flips Active = No on every INPUT - House Values row for that house plus the mirror SYS - House Assets row; history and the HOUSES - {House} sheet stay in place. Activity Type is House Expenses; sub-label is Tracking stopped; Amount shows —.
  • investment_add — Create an investment account from Assets → Investments → Add new. Adds rows to INPUT - Investments (current year block) and SYS - Assets. Activity Type is Investment; Action sub-label is Account added. Amount shows the starting value (0 if no starting value was supplied).
  • investment_deactivate — Stop tracking on an investment account from Assets → Investments → Update. Flips Active = No on every INPUT - Investments row for that account plus the mirror SYS - Assets row; month history stays in place. Activity Type is Investment; sub-label is Tracking stopped; Amount shows —.
  • debt_add — Create a debt account from Planning → Debts → Add new. Adds one row to INPUT - Debts with Active = Yes. Activity Type is Debt; Action sub-label is Account added. Amount shows the required opening balance entered on the form (may be $0.00).
  • debt_deactivate — Stop tracking on a debt from Planning → Debts → Update. Flips Active = No on the matching INPUT - Debts row (the row is kept for history). The debt drops out of the planner, Rolling Debt Payoff, Bills Due, and active selectors. Activity Type is Debt; sub-label is Tracking stopped; Amount shows —.

Remove button

  • Enabled only for Donation rows until safe undo exists for other event types.
  • For donations: after confirmation, the log line is removed; if INPUT - Donation still has a matching row (charity, date, amount, tax year, payment type, comments), that sheet row is removed too.
  • For other events (quick_pay, bill skip, house expense, bank account add, bank_account_deactivate, house_add, house_deactivate, investment_add, investment_deactivate, debt_add, debt_deactivate, …) the button is greyed out — delete those lines directly on LOG - Activity if needed.

Filters

  • Date range — Applies to when the row was logged (server timestamp).
  • Payee — Substring match, case-insensitive.
  • Type — Lists every activity type present anywhere in LOG - Activity, derived the same way as the Type column.
  • Amount min / max — Optional currency-style inputs; filter on the logged amount.
  • Up to 500 matching rows load per Apply; the table shows 20 at a time with Previous / Next.

Table columns

  • Logged at — When the script appended the row (browser locale).
  • Payee — The Cash Flow / bill payee for that action.
  • Type — A short label derived for scanning: Loan, Bill, HOA, Tuition, Income, Donation, Upcoming, Bank, Investment, Repair / Maintenance / Utilities / Property Tax (from the House Expenses Type), House Expenses if that field was blank, or Other.
    • It is derived, not a column you edit. Payees are matched to INPUT - Debts (Account Name + Type) and INPUT - Bills (Payee + Category); keyword checks cover HOA and tuition; donation events map to Donation; house_expense events use the log Category.
    • Under the pill, a small Action sub-label disambiguates events that share a Type (e.g. the Bill pill can show Bill added, Tracking stopped, Bill skipped, or Bill autopay).
    • The Type filter operates on the broad pill, so filtering by Bill still surfaces every bill lifecycle event.
  • Amount — The amount stored for the event (payment amount, or 0 for a skip). Lifecycle events that do not move money — Tracking stopped (bill_deactivate, bank_account_deactivate, house_deactivate, investment_deactivate, debt_deactivate) — show — instead of $0.00. Account creation events (bank_account_add, investment_add, house_add, debt_add) show the supplied opening / starting / current value, or $0.00 when none was provided.
  • Bill due date — The obligation date for that cycle (from the log entry date). Not necessarily the day you clicked the button.

Sorting

  • Click a column header to sort; click again to flip ascending/descending.
  • Sorting applies to all rows that matched your filters on the last Apply (up to 500) — not only the current page.

Source of truth

Cash Flow, Debts, and the other INPUT/SYS tabs remain authoritative for balances. The Activity page helps you audit what the automation recorded; open LOG - Activity directly for the raw grid or to copy rows.

Bills

The Bills page (on Cash Flow) has two internal views: Due this period for execution, and Manage bills for the underlying list on INPUT - Bills.

Due this period

  • Three groups: Overdue, Next 7 Days, and Recurring Bills (No Due Date).
  • Each card shows the bill name, amount, and due date (when applicable), plus Pay and Skip.
  • The card does not list category, autopay, or “varies” flags — those remain on the sheet for your reference.

What the buttons do

  • Pay — Opens Quick add with payee, due date, and amount pre-filled. You can change the amount before saving. The amount shown on the card is the same number Pay starts with; there is no separate “suggested” line.
  • Skip — Marks the bill skipped (per your backend rules) and refreshes the lists. Typically writes 0 into the matching Cash Flow cell.

Where amounts come from

  • Rows from INPUT - Bills use that sheet’s bill fields.
  • Items tied to INPUT - Debts (minimum payments) use the planner / debt amount.

Default Amount = $0

  • Bills can use 0 for variable or “enter when known” expenses. They still appear in Overdue / Next 7 Days when the Cash Flow cell for that cycle is empty.
  • Pay opens Quick add with 0 — set the real amount there.
  • Autopay never writes a payment when the default amount is 0 (even if Autopay is Yes and Varies is No).

Recurring (no due date)

  • Bills not yet tied to a dated row in INPUT - Bills land in the third section.
  • Pay opens Quick add with today’s date and amount 0 so you can enter the real payment.
  • Map or add due dates on INPUT - Bills when you want them in the dated lists.

Manage bills

  • Lists every row from INPUT - Bills with sortable columns (Payee, Due Day).
  • Default sort is Due Day, ascending. Click a column header to sort; click again to flip direction.

Add bill

  • Appends a row to INPUT - Bills. Category is required (both client and server side).
  • Logs bill_add on LOG - Activity.

Stop tracking

  • Sets Active = No on the bill row. The row is kept for history; no payments are reversed.
  • Logs bill_deactivate (sub-label Tracking stopped, amount —).

Status & errors

  • Feedback from Pay / Skip and from Add bill / Stop tracking appears in the status line under the Bills heading, not next to Run Planner. Success messages auto-clear.
  • If the bills queue fails to load, the error shows under the Bills heading and in the Overview Bills Due card summary.

Properties

  • House Expenses — Adds rows to the per-property HOUSES - … tabs. The house selector refreshes every time you open the tab, so houses created via Assets → House Values → Add new appear immediately (their HOUSES - {House} sheet is created automatically at add time).
  • Property performance — See the next section for rent, expenses, and how the Type column on SYS - House Assets ties them together.

Property performance (detail)

Active houses only

  • The table and portfolio mini-cards show active houses only — houses whose SYS - House Assets → Active is blank (legacy) or anything other than No / n / false / inactive.
  • Stop-tracked houses drop out on the next load/refresh. Their INPUT - House Values rows, SYS - House Assets row, and HOUSES - {House} sheet are preserved; only the view is filtered.
  • Portfolio totals (Equity, Rent, Expenses, Net) are summed from the same filtered row set, so cards and rows always agree.

Rent

  • Sums Income rows on INPUT - Cash Flow for the selected year.
  • Only counts rows whose Payee starts with Rent followed by the same house name as in SYS - House Assets — for example, Rent Oakley House - ….

Rent only for rentals

  • The Type column on SYS - House Assets controls this.
  • Rental (and the common typo Renal) use Cash Flow rent.
  • Home and Vacation Home show $0 rent even if Income rows exist.
  • A blank Type still sums rent (legacy behavior).

Expenses

  • Come from each property’s HOUSES - … sheet.
  • The table sums Cost + Service Fees for the selected year on the sheet whose location matches the house (matching is case- and spacing-insensitive).

HOUSES sheet

  • There must be a HOUSES - … tab aligned with the SYS House name. Assets → House Values → Add new creates this sheet automatically with the canonical header layout and column widths copied from an existing HOUSES tab.
  • If no matching HOUSES sheet exists — or the name differs (e.g. SD Condo vs San Diego House) — expense totals stay $0 until you add or rename the tab.

Net

  • Net = Rent − Expenses for that row.
  • The portfolio mini-cards sum the same figures across all houses.

Planning

Planning is the decision layer — it answers "what should I do next?" by interpreting data from Bills, Upcoming, Debts, Bank Accounts, and Cash Flow. It does not store its own copies of those numbers and does not move money. The single payment path remains Cash Flow → Quick Add.

The landing view is Next Actions (featured at the top). Everything else is a Planning tool — deeper views you drill into from a Next Actions card:

  • Next Actions — Prioritized short lists of what to pay, finish, or review now. See the next section for details.
  • Debts — Edit INPUT - Debts; planner output uses the balances and minimums there.
  • Retirement and Purchase Sim — Use the latest snapshot plus your inputs. Details depend on sheet configuration.
  • Debt Overview vs Rolling Debt Payoff:
    • Debt Overview — Read-only reference view of your debt structure.
    • Rolling Debt Payoff — The monthly action decision engine. See its own section below.

Debts — Update / Add new / Stop tracking

  • Update saves a single field on the selected debt row in INPUT - Debts (Account Balance, Due Date, Credit Limit, Minimum Payment, Credit Left, Int Rate). Saving recomputes Acct PCT Avail and reruns the debt planner. The TOTAL DEBT summary row is always excluded from the selector.
  • Add new inserts one row directly above the TOTAL DEBT summary on INPUT - Debts with Active = Yes. All fields are required: Account name, Type, Account balance, Minimum payment, Credit limit, Interest rate %, and Due day of month. Enter 0 for a field that does not apply (for example, Credit limit on a Loan or HELOC). Credit Left is derived (Credit limit − Account balance) — not a separate input — and Acct PCT Avail is recomputed from it.
  • Type is a dropdown populated from the distinct Type values currently on INPUT - Debts (e.g. Credit Card, Loan, HELOC). Pick Other… to reveal a text input and add a new type on the fly — the typed value is written as-is to the new row's Type cell and becomes available in the dropdown for future adds.
  • Cash Flow auto-seed — Also writes a matching Expense row on the current year's INPUT - Cash Flow tab so the new debt shows up in Bills Due, Upcoming, and the planner email without you editing the sheet. The row is inserted in the Expense block with Type = Expense, Payee = Account name, Active = YES, and Flow Source inferred from Type (Credit Card → CREDIT_CARD, everything else → CASH). All month cells are left blank — nothing has been paid yet. The first time you run Quick Add → Pay or Bills → Pay for this debt, that action fills the correct month cell; until then Bills Due treats the current cycle as unhandled. Idempotent: if an Expense row for the same payee already exists on that year tab (e.g. pre-seeded by hand, or the name was previously stop-tracked and re-added), the existing row is left untouched. If the current-year Cash Flow tab does not exist yet, the seed is skipped and the status line explains why.
  • Duplicate names are rejected across every non-summary row on INPUT - Debts, including rows you previously stop-tracked, so names stay reserved. Reserved summary names (e.g. TOTAL DEBT) are always rejected. Inline errors show next to the Account name field and focus the input.
  • Status text during the flow is Creating debt account…. After success, the panel flips back to Update and preselects the new debt.
  • Add new logs debt_add to LOG - Activity. Activity Type is Debt; Action sub-label is Account added. Amount shows the required opening balance (which may be $0.00 if you entered 0). It does not re-run the full planner — use Run Planner + Refresh Snapshot when you want projections and Overview updated.

Debts — Stop tracking (soft deactivate)

  • Update mode shows a secondary Stop tracking button next to Save Debt Update. Use it when a debt is closed / paid off / no longer relevant without losing history.
  • Clicking it opens an inline confirmation. On confirm, the matching INPUT - Debts row is flipped to Active = No. The row is never deleted; balance, minimum payment, credit limit, interest rate, due day, and the name stay intact so historical quick_pay / bill matches still classify correctly.
  • Inactive debts drop out of the Planning Debts selector, Debt Overview, Rolling Debt Payoff (waterfall, HELOC gate, focus/next-debt selection, payment windows), Bills Due, Overdue / Next 7, and Quick add Flow Source inference. They are also excluded from Debt totals and high-utilization alerts.
  • Historical LOG - Activity classification is not filtered by Active — rows that used to hit a now-inactive debt still show the right Type and Kind so you can audit past payments.
  • Duplicate-name validation still blocks reuse of a stop-tracked debt name when adding a new debt (names stay reserved).
  • Status text during the flow is Stopping tracking….
  • Backward compatibility: If the Active column is missing on your workbook, the script self-heals — it adds the column automatically the first time you Add new or Stop tracking. Existing rows with blank Active are treated as active, and if no Active column exists at all the legacy fallback still applies (balance > 0 || minimum payment > 0).
  • Logs debt_deactivate to LOG - Activity. Activity Type is Debt; Action sub-label is Tracking stopped; Amount shows — because no money moved.
  • New Yes / No values written into the Active column inherit the row's text style so they visually match the rest of the row.

Next Actions

Your primary Planning entry point. Next Actions reads your existing data (Bills, Upcoming remaining balances, Debts, Bank Accounts) and returns a prioritized short list in three sections. It does not move money — each card is a pointer you can open to the relevant tool.

What it shows

  • Summary row — three figures:
    • Cash to use — What's safely available right now across eligible Bank Accounts, after each account's minimum buffer. Credit cards, accounts flagged do-not-touch, and inactive rows are excluded.
    • Due soon — Sum of every Urgent item (overdue + due in the next 7 days).
    • Available after urgent — Cash to use minus Due soon. This is what's left to suggest moving beyond your must-pays.
  • Urgent — Overdue items and anything due in the next 7 days across Bills, debt minimums, and Upcoming obligations. Overdue items come first; after that, larger near-term obligations (roof, solar, big tax payments) are ranked ahead of tiny bills so they are never buried. If Cash to use is less than Due soon, a Review cash gap row appears at the top explaining the shortfall. The top obligations are shown individually; the remainder collapses into a single Other bills due soon row with a count and total so the visible amounts still reconcile with the summary.
  • Recommended — Suggested next moves once Urgent is covered. Two kinds of suggestion:
    • Non-urgent Upcoming items due 8–30 days out, with earlier / larger items surfaced first. "Pay now to keep it out of the urgent window."
    • One Apply $X to <debt> suggestion, sized from a conservative slice of Available after urgent and capped at the debt's remaining balance. The target and APR details are confirmed on Rolling Debt Payoff — Next Actions keeps the reason short ("Confirm in Rolling Debt Payoff") to avoid duplicating that surface's Focus-debt narrative.
  • Optimize — Sparse by design. Only appears when there is genuine optimization signal (e.g. extra debt acceleration when no spare cash is available right now). Usually empty. HELOC strategy is not surfaced here — it lives on the Rolling Debt Payoff "HELOC strategy" card.

Open links — where each card routes

Every card is clickable. The Open … link on the right (also the whole card's click target) routes to the deeper tool:

  • Open Bills → Cash Flow → Bills Due
  • Open Upcoming → Cash Flow → Upcoming
  • Open Payoff → Planning → Rolling Debt Payoff
  • Open Debts → Planning → Debts

The Review cash gap row is informational and does not open anywhere — its reason line explains the shortfall.

Guardrails (what Next Actions will not do)

  • No writes. No payments. Quick Add remains the single payment path.
  • No forecasts. Cash to use is a conservative current-state number — it ignores future income, pending transfers, credit lines, and investments.
  • No recommended actions while a cash gap exists — must-pays come first.
  • Every suggestion is explainable in one sentence from current data.

Debt Overview

What it is

  • Read-only reference view of your debt structure: balances, minimums, APRs, and estimated payoff at current minimums.
  • Not an action planner — no recommendations, no cash-flow notes, no decision content. For this month's action plan, use Rolling Debt Payoff. For what to do next across bills / upcoming / debts, start at Next Actions.

Data sources

  • Balances and minimums — From your Debts (active rows only — stop-tracked debts are hidden from the bar chart, table, and totals).
  • CF paid — Sums expense rows on Cash Flow for the last two calendar years where the payee matches the debt.

Estimated payoff

  • Loan and HELOC — Monthly amortization loop using the sheet’s APR and minimum: interest on remaining balance, then principal.
  • Other debt types — Rough estimate as balance ÷ minimum payment.
  • If the minimum is not greater than monthly interest on a loan, payoff is shown as unavailable.

Summary

The summary row shows only Total debt balance and Total monthly minimums. Decision-style context (safe-to-use cash, projected cash flow, highest-APR callouts, longest-payoff callouts) intentionally lives elsewhere — on Next Actions and Rolling Debt Payoff — so Debt Overview stays a pure reference view.

Rolling Debt Payoff

This is the monthly decision engine — separate from Debt Overview (which is a read-only reference). The anchor month is the latest Cash Flow month that has any income or expense data — not today’s calendar date.

Standard view

The tab opens in a compact decision layout. Everything here is driven by the same planner JSON; the Show details toggle at the bottom expands the full forecasts, audits, and diagnostics.

Cash to use now

  • Single input at the top. Type how much you want to deploy this month.
  • The planner caps it at Safe to use (your deployable max after reserves, buffers, and holds) and reruns the waterfall instantly.
  • A short live formula line shows min(Cash to use now, Safe to use).

Decision card

A three-line narrative summary:

  • Recommendation — What the plan suggests (e.g. deploy $X now, focus on Account Y).
  • Why — The short reason (highest-APR target, balance caps, or a strategy choice).
  • Caution — Any near-term risk or reason the recommendation is conservative.

HELOC strategy card

  • Appears when a HELOC is modeled.
  • Shows the recommended action and the draw size.
  • What would change this? lists the actionable levers (HELOC APR, eligible-debt spread, upcoming expense hold, recurring monthly repayment) with current values, so you can see what’s driving the recommendation.

Payment result — what the numbers mean

A compact table of the accounts the plan actually touches this month:

  • Account — The debt name plus its role in the waterfall:
    • Small balance — paid first to close small accounts.
    • Focus debt — the highest-priority target this month.
    • Next debt — spill once Focus debt is capped.
    • Excess — anything else (e.g. a low-rate line kept for last).
  • Action — Paid off (this month) when the plan clears the balance (shown green), or Partially paid when it reduces the balance.
  • Remaining — Projected balance after this month’s payment.
  • [Add payment] — Small pill on rows with a real payable amount. Clicking it jumps to Cash Flow → Quick add pre-filled with the payee and the planner’s amount; review or adjust, then save. Hidden in standalone contexts where Quick add is not available.

Why not more?

  • Shown when the plan deploys less than Safe to use.
  • Explains the two holds that caused it: near-term planned hold and unmapped card risk hold.
  • Each bullet has an inline [View] pill that opens Cash Flow → Upcoming when the hold is driven by planned upcoming expenses, so you can act on the underlying rows.

Strategy control

  • Small toggle to switch between the conservative default and a more aggressive concentration (e.g. push a larger share of the remainder onto Focus debt).
  • The decision card and payment result update in place.

Safe to use (liquidity model)

Derived from your Accounts. The legacy $100k / $100k hardcoded reserve and buffer constants are retained for audit only and no longer drive the numbers.

  • Total cash — Full liquid total across your Accounts (all cash + checking + savings).
  • Reserve — Sum of current balances for accounts flagged do-not-touch.
  • Buffer — Sum of per-account minimum buffers for eligible, non-reserve accounts.
  • Cash available to use — Total cash − Reserve − Buffer, before subtracting holds.
  • Near-term planned hold — Cash reserved for upcoming planned expenses within 90 days. First driver behind Why not more?.
  • Unmapped card risk hold — Safety hold for card-funded planned expenses not yet mapped. Second Why not more? driver; when fed by Upcoming rows, the bullet exposes the same [View] action.
  • Safe to use (Deployable max) = Total cash − Reserve − Buffer − Near-term hold − Unmapped card risk hold (floored at 0).
  • Cash to use now — Your planning input.
  • Execute now = min(Cash to use now, Safe to use). This is the value the waterfall uses. The legacy $50k monthly cap has been retired — your input is the limit.
  • Future / Conditional Cash (not available yet) — Variable-income-contingent allocations surfaced below execution totals. Not part of Execute now until income arrives.

Show details — planner mechanics

Show details expands the full output: cash-bridge audit, allocation audit (month 0), 12-month cash table, yearly projection, forecasts, diagnostics, and raw JSON. Click Hide details to collapse back.

  • Cash out vs minimums — Required cash out uses Cash Flow expense lines. Debt minimums appear as reference only in triggers and JSON; they are not double-counted on top of Cash Flow.
  • Income split — Stable income = Cisco (payee contains “Cisco”) plus the fixed recurring rent payees in product rules. Other income is variable; variable inflows use the 50% / 30% / 20% split (debt / reserve+cash / flexible).
  • Forward months — After the anchor month, stable income uses trailing medians. Forward expenses use a recurring operating baseline only: expenses are bucketed (recurring operating / debt cleanup / irregular) and future months sum trailing medians (6 → 3 → 1 non-empty fallback per payee) for recurring_operating payees only — not total cash-out. High-volatility credit-card payees go to debt cleanup; taxes, repairs, and similar keywords go to irregular. Variable income in projections defaults to $0 (conservative).
  • Debt ordering — Extra principal (after the reserve rule) goes to credit cards by highest APR first, then HELOC, Tahoe, San Diego, Missouri, and San Jose last. Loan Depot - San Jose House and any debt with APR ≤ 2.25% get LOW_RATE_KEEP_LAST on the normalized debt row.
  • Role names in the waterfall — Presentation labels only; math and JSON field names are unchanged.
    • Small balance → engine’s Cleanup bucket.
    • Focus debt → Primary.
    • Next debt → Secondary.
    • Excess → Overflow.
  • Confidence — overall_plan_confidence is a simple rubric (see JSON overall_plan_confidence_rubric); credit-card “spend” without statement-balance tracking is marked LOW.
  • Operating vs forecast — The Standard view is the operating picture (this month’s decision). The 12-month cash table, yearly rows, and next-three-month preview under Show details are forecasts, not a month-by-month instruction list.
  • Plan status:
    • OK — Core forward model is clean.
    • PARTIALLY_VALID — An execution-integrity check failed (cash split vs execution total, missing per-account lines when cash extra is non-zero, or total executable mismatch). Advisory noise does not change status.
    • NEEDS_REVIEW — Sheet triggers fire.
    • INVALID — Income/expense forward numbers cannot be trusted.
  • JSON highlights — this_month_plan, action_decision_box, next_3_month_preview, summary (anchor month as yyyy-MM), key_warnings, next_12_months, yearly_projection (with debt_reduction_vs_prior_year = prior year ending debt minus this year ending), debt_priority, property_watchlist, assumptions, confidence, diagnostics (including expense_baseline irregular flags).

Debt Planner email

Running the debt planner may send a short email titled Debt Planner Update. It is a quick glance — bill buckets first, then a compact summary (stability, net worth, cash flow, extra-payoff hints). Full definitions live here, not in the email.

What you see

  • Overdue bills — Same idea as Bills → Overdue: due date is before today and the matching Cash Flow month cell for that cycle does not yet count as having a value. Combines INPUT - Debts and INPUT - Bills.
  • Pay now — Scheduled amounts due in the next 7 days from the run date: INPUT - Debts minimums (with the “handled this month” rule below) plus dated rows from INPUT - Bills whose Cash Flow cell for that cycle is still empty. If the same payee + due date appears as both, the debt line is used once — no double count.
  • Pay soon — Same combined sources, for due dates day 8 through day 30 out.
  • Recommended total to pay now — Appears only when the planner suggests an extra payment on top of Pay now minimums. Otherwise, Pay now covers minimums; the summary section still notes the best card target and liquidity when relevant.

“Already paid this month” rule

  • If the expense row for a debt already has a value in the current month column on Cash Flow (same “handled” rule as Bills), that debt is omitted from Pay now / Pay soon for that run.
  • Re-run the planner after updating Cash Flow to refresh the email.

Why it matches the Bills page

The Bills page uses the same underlying queues. Next 7 Days on the web is calendar-based through seven days; Pay now in the email uses the same 7-day window from the run date, so counts usually line up.

Sheet names (quick reference)

INPUT sheets

  • INPUT - House Values — Year blocks with monthly valuations per house.
  • INPUT - Cash Flow {year} — Monthly columns; Rent payees drive rental income.
  • INPUT - Bills — Dated bills (category, autopay, varies, payment source, etc.). Drives most Bills cards.
  • INPUT - Upcoming Expenses — Planned expenses. Edited from Cash Flow → Upcoming.
  • INPUT - Debts — Loan accounts; minimums can surface on the Bills page.
  • INPUT - Donation — Charitable giving log organized by tax-year blocks. Added from Cash Flow → Donations.
  • INPUT - Bank Accounts — Per-year balance blocks. Written by Assets → Bank Accounts.
  • INPUT - Investments — Per-year balance blocks for investment accounts (Account Name, Type, Active, Jan..Dec, totals / delta). Canonical active/inactive state for investments. Written by Assets → Investments.

SYS sheets

  • SYS - House Assets — One row per property: House, Type, Loan Amount Left, Current Value, Active.
  • SYS - Accounts — Liquid accounts with use-policy flags; feeds the Safe-to-use liquidity model.
  • SYS - Assets — One row per investment account: Account Name, Type, Current Balance, Active. Mirrors the latest current-year value from INPUT - Investments.

HOUSES sheets

  • HOUSES - {House} — Expense detail per property. Name should match the SYS house for rollups. Created automatically by Assets → House Values → Add new with canonical headers (Item, Type, Date, Location, Cost, Service Fees Paid, Insurance covered, Payments Links, Notes) and column widths copied from an existing HOUSES sheet.

Output & audit

  • OUT - History — Planner run snapshots (cash, debt, net worth).
  • LOG - Activity — Audit log for quick_pay, bill skip / autopay / add / deactivate, house expense, donation, upcoming actions, bank account add, house add / deactivate, and investment add / deactivate. Filter, sort, paginate, and remove (Donation-only) mistaken rows from the Activity page.
